Como surgiu a Linguagem C?

A linguagem C surgiu em 1972 como uma evolução de outra linguagem criada por Ken Thompson, chamada B. Inicialmente, não existia um padrão único para a linguagem, o que fazia com que diferentes organizações utilizassem implementações distintas. Com o tempo, essas organizações se uniram para definir um padrão oficial para o C, garantindo maior compatibilidade entre compiladores e sistemas.

Por que programar em C?

- COBOL foi criada para o processamento de registros e aplicações comerciais.
- Pascal foi desenvolvida com foco no ensino de programação.
- FORTRAN foi projetada para cálculos científicos.
- LISP e PROLOG foram desenvolvidas para aplicações na área de Inteligência Artificial.

Diferentemente dessas linguagens, C é extremamente versátil e pode ser utilizada em praticamente qualquer tipo de projeto, desde sistemas operacionais até compiladores.

A linguagem oferece desempenho próximo ao Assembly, além de ser simples, portátil, popular e modular. Com ela, também é possível manipular diretamente a memória por meio dos endereços dos objetos. O C permite o uso de macros e fornece grande liberdade ao programador para implementar soluções da forma que considerar mais adequada.

Filosofia do C

Em C, é mais fácil implementar módulos independentes, seguindo uma abordagem semelhante à componentização, o que contribui para a qualidade e manutenção de cada parte do sistema.

Essa filosofia foi fortemente influenciada pelo UNIX, cujo princípio é fazer uma tarefa de cada vez e fazê-la bem, permitindo que pequenos componentes trabalhem juntos para atingir objetivos maiores.

C vs C++

O C++ pode ser visto como uma extensão da linguagem C, adicionando recursos como classes, programação orientada a objetos, sobrecarga de operadores, templates e diversas outras funcionalidades.

Embora mantenha grande compatibilidade com C, o C++ é uma linguagem consideravelmente mais ampla e complexa.

Ciclo de Desenvolvimento

1. Escrever o código-fonte

O primeiro passo é escrever o programa em um arquivo com extensão .c.

2. Compilar o programa

A compilação verifica se o código está correto. Durante esse processo, o compilador pode detectar erros e emitir avisos (warnings) sobre possíveis problemas.
Se não houver erros, será gerado um arquivo-objeto (.obj em alguns sistemas).

Exemplos de compilação: 

 

// Turbo C
tcc programa.c

// Unix
cc programa.c

3. Realizar a linkagem

Nesta etapa, o executável é criado a partir dos arquivos-objeto e das bibliotecas utilizadas pelo programa.
Em sistemas DOS, as bibliotecas costumam possuir extensão .LIB, enquanto em sistemas Unix a extensão mais comum é .out
O responsável por essa fase é o Linker, que reúne todos os arquivos-objeto em um único executável. A linkagem pode falhar caso alguma função, biblioteca ou símbolo necessário não seja encontrado.
Para definir o nome do executável gerado, utilize a opção -o. Caso ela não seja especificada, o compilador geralmente criará um executável chamado a.out. 

 

gcc program.c -o programa

4. Executar o programa

Após a geração do executável, basta executá-lo pelo terminal. 

 
./programa.out
Se o programa não funcionar corretamente, o problema provavelmente está na lógica implementada pelo desenvolvedor ou em algum erro que não foi detectado durante a compilação.
